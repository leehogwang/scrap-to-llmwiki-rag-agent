ClipWiki 시스템 아키텍처

브라우저 스크랩, Function Calling 기반 위키 생성, Graphify 시각화를 하나의 축적형 RAG 워크플로우로 통합한 시스템 구조

1. 사용자 상호작용

  • 웹 페이지에서 Alt+Drag 기반 스크랩 수행
  • 좌측 패널을 통한 질의응답 수행
  • 위키 생성·갱신 및 Graphify 탐색 수행
입력 시작점

2. 크롬 익스텐션

  • DOM, 이미지, 유튜브 카드 요소를 감지
  • 스마트 스크랩 후보 청크를 수집
  • 캡처 payload를 서버로 전송
브라우저 계층

3. 앱 서버

  • 입력 검증 및 moderation 절차를 수행
  • Function Calling 루프를 관리
  • Codex Auth 또는 OpenAI API를 호출
제어 계층

4. 데이터·도구 계층

  • SQLite에 scrap, wiki, graph cache를 저장
  • Notion API를 통해 위키와 스크랩을 게시
  • YouTube transcript와 OCR 결과를 병합
외부 연동

5. 생성 파이프라인

  • 신규 스크랩을 기반으로 위키를 생성하거나 보강
  • TF-IDF 기반 스마트 스크랩 확장을 수행
  • Graphify를 재구축하고 놀라운 연결을 추론
추론 계층

6. 사용자 경험

  • 스크랩 축적에 따라 강화되는 RAG 구조 제공
  • 위키, 그래프, 질의응답 간 순환 구조 제공
  • 단발성 검색을 넘어서는 개인 지식층 형성
출력 계층
Developer-in-the-Loop
모델은 함수 스키마를 바탕으로 도구 호출을 제안하며, 서버는 이를 검증·실행한 후 결과를 다시 모델에 전달합니다. 사용자는 스크랩, 위키, Graphify를 통해 지식을 축적하며, 축적된 결과는 이후 질의의 RAG 품질을 점진적으로 향상시키는 피드백 루프를 형성합니다.
